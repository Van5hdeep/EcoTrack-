# EcoTrack

EcoTrack is a modern web application designed to help users discover tree plantation drives, join environmental movements, and track their carbon impact. Built with an emphasis on premium design and intuitive user experience, EcoTrack serves as a platform for communities to collaborate on sustainability initiatives.

## Features

- **Plantation Drives Search**: Find nearby events filtered by city and organizer.
- **Impact Dashboard**: Visualize your personal environmental contribution, including total trees planted and CO2 offset.
- **Achievement System**: Earn badges based on your sustainability milestones.
- **Community Stories**: Read and share personal experiences from plantation volunteers.

## Tech Stack

This project was built with a modern frontend stack to ensure performance and maintainability:

- **React** - Component-based UI development
- **Vite** - Next-generation frontend tooling for rapid development
- **Tailwind CSS** - Utility-first CSS framework for custom styling
- **Framer Motion** - Production-ready animation library
- **Recharts** - Composable charting library built on React components
- **React Router** - Declarative routing for React web applications
- **Lucide React** - Beautiful, consistent iconography

## Getting Started

### Prerequisites

Ensure you have Node.js and npm installed on your local machine.

### Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/Van5hdeep/EcoTrack-.git
   ```

2. Navigate into the project directory:
   ```bash
   cd EcoTrack
   ```

3. Install the dependencies:
   ```bash
   npm install
   ```

### Running the Development Server

To start the local development server:

```bash
npm run dev
```

The application will be accessible at `http://localhost:5173`.

### Building for Production

To create an optimized production build:

```bash
npm run build
```

This will generate a `dist` folder containing the compiled assets.

## Project Structure

- `/src/components` - Reusable UI components (Cards, Navbar, Footer, etc.)
- `/src/pages` - Main application views (Home, Drives, Dashboard, Stories)
- `/src/data` - Local mock data simulating an API response
- `/src/styles` - Global CSS styles and Tailwind configurations
